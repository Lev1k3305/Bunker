# 🎲 БУНКЕР — электронный помощник для настольной игры

## Обзор проекта
- **Название**: БУНКЕР
- **Цель**: цифровой помощник ведущего для популярной социальной настольной игры «Бункер» — генерирует катастрофу, параметры убежища и секретные досье игроков с узкими профессиями, а также ведёт учёт раундов, раскрытия характеристик и голосований.
- **Особенности**:
  - **Лобби**: перед стартом каждый игрок вписывает своё имя, ведущий подробно проговаривает правила игры (6 пунктов: катастрофа/бункер, досье, раунды, ситуации, голосование, победа)
  - 80+ узкоспециализированных профессий (хирург-трансплантолог, инженер-атомщик, снайпер спецназа и т.д.)
  - 16 детально прописанных глобальных катастроф (ядерная война, зомби-апокалипсис, вторжение инопланетян и др.)
  - Случайные параметры бункера (площадь, срок пребывания, этажность, доп. помещения, запасы провизии)
  - Полные карточки персонажей: пол/возраст, здоровье, фобия, хобби, черты характера (+/−), инвентарь, доп. информация
  - Механика поэтапного раскрытия характеристик по раундам (клик по строке — открыть/скрыть)
  - **Приватный просмотр «своей» карточки** — кнопка с иконкой глаза открывает модальное окно с полностью раскрытым досье (профессия + все характеристики) только для владельца карточки, не влияя на то, что видно остальным
  - Переброс скрытых характеристик отдельного игрока
  - **25 развёрнутых «Ситуаций»** для озвучивания вслух в любой момент партии (моральные дилеммы, внешний контакт, технические сбои, внутренние конфликты, находки, угрозы) — кнопка «Ситуация» в игровой панели
  - **Голосование за исключение**, которое разблокируется только начиная с 3-го раунда: подсчёт голосов по каждому выжившему (+/−), автоматическое определение лидера голосования, обработка ничьей с ручным выбором
  - Исключение/восстановление игроков с визуальным индикатором выживших
  - Случайные события каждого раунда (короткие карточки, всплывают автоматически при переходе в новый раунд)
  - Сохранение состояния игры в localStorage (можно закрыть вкладку и продолжить)
  - Тёмный постапокалиптический дизайн: текстура шума, виньетка, ржавые акценты, анимации карточек

## API-эндпоинты
| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/api/game/new` | Тело: `{ playerCount }`. Генерирует катастрофу, параметры бункера и полный набор персонажей |
| `POST` | `/api/game/reroll` | Тело: `{ field }`. Возвращает новое случайное значение для указанного поля характеристики |
| `GET` | `/api/game/event` | Возвращает случайное событие текущего раунда |
| `GET` | `/api/game/situation` | Возвращает случайную развёрнутую «ситуацию» для озвучивания вслух |
| `GET` | `/api/game/catastrophe` | Возвращает новую случайную катастрофу |
| `GET` | `/api/game/bunker` | Возвращает новые случайные параметры бункера |

## Архитектура и данные
- **Backend**: Hono (TypeScript) на Cloudflare Workers/Pages
- **Frontend**: Vanilla JS (SPA без сборки фреймворка) + Tailwind-free кастомный CSS, Axios для запросов к API, Font Awesome для иконок, Google Fonts (Oswald + Rubik)
- **Данные игры**: `src/data.ts` — статические списки профессий/здоровья/фобий/хобби/черт/инвентаря/доп.инфо, катастроф, параметров бункера и событий раундов; функции генерации без повторов внутри одной игры
- **Хранение прогресса**: `localStorage` на клиенте (ключ `bunker_game_state_v1`) — данные не уходят на сервер, что подходит под serverless-ограничения Cloudflare Pages
- **Персистентность на сервере**: не используется (не требуется для данного приложения — вся партия хранится в браузере ведущего)

## Как пользоваться
1. На стартовом экране выберите количество выживших (4–16) ползунком и нажмите «Далее: лобби».
2. **Лобби**: ведущий зачитывает подробные правила (6 пунктов), а каждый игрок вписывает своё имя в соответствующее поле.
3. Нажмите «Начать катастрофу» — сгенерируется катастрофа и параметры бункера. При желании перегенерируйте их отдельно кнопками «Другая катастрофа» / «Другой бункер», либо вернитесь в лобби.
4. Нажмите «Спуститься в бункер» — откроются карточки всех игроков с профессией (видна сразу) и скрытыми характеристиками.
5. Каждый игрок вслух называет свою профессию. Дальше по раундам ведущий (или сами игроки) кликают по нужной характеристике в своей карточке, чтобы раскрыть её остальным.
6. **Своя карточка**: каждый игрок в любой момент может нажать иконку глаза на своей карточке — откроется приватное модальное окно со всеми характеристиками сразу, видимое только ему (остальным игрокам при этом ничего не раскрывается).
7. Кнопка «Ситуация» в верхней панели выводит случайную развёрнутую ситуацию для озвучивания и обсуждения — можно использовать в любой момент, независимо от раунда.
8. Кнопка «Следующий раунд» увеличивает счётчик раунда и показывает случайное короткое событие бункера.
9. **Голосование**: кнопка «Голосование» в топ-баре заблокирована (значок замка) до 3-го раунда включительно. С 3-го раунда она активируется — открывается окно с постатейным подсчётом голосов (+/−) за каждого выжившего; после подтверждения исключается набравший больше всего голосов (при ничьей — отдельное окно для ручного выбора).
10. Также можно вручную нажать иконку «исключить» на карточке — она визуально помечается как «ИСКЛЮЧЁН», а индикатор выживших обновляется.
11. Пока характеристика скрыта, доступна кнопка переброса (кубик) — на случай, если ведущий хочет заменить нераскрытый параметр.
12. Кнопка «Новая игра» (с подтверждением) сбрасывает партию и возвращает на стартовый экран.
