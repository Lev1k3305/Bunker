# 🎲 БУНКЕР — сетевая настольная игра на выживание

## Обзор проекта
- **Название**: БУНКЕР
- **Цель**: полноценная сетевая версия популярной социальной игры «Бункер». Реальные игроки заходят каждый со своего устройства по короткому коду комнаты, выбирают своё место, получают приватное досье (характеристики видны только им, пока они сами их не раскроют), проходят раунды раскрытия, обсуждают ситуации в общем чате и голосуют за исключение с ограничением по времени.
- **Особенности**:
  - **Комнаты с кодом**: любой может создать комнату (4–16 мест) и получить 5-символьный код; остальные подключаются по этому коду каждый со своего устройства
  - **Выбор своего места**: в лобби каждый сам занимает свободное место и вписывает своё имя — так определяется, кем он будет в игре
  - **Настоящая приватность на сервере**: в начале игры характеристики видит только их владелец; остальным игрокам они не передаются сервером вовсе, пока владелец сам их не раскроет (не «локальный peek», а честная серверная фильтрация на каждый запрос)
  - **Хост комнаты** — первый занявший место игрок; управляет катастрофой, бункером, раундами, ситуациями и голосованием
  - 80+ узкоспециализированных профессий, 16 детально прописанных катастроф, случайные параметры бункера
  - Полные карточки персонажей: пол/возраст, здоровье, фобия, хобби, черты характера (+/−), инвентарь, доп. информация
  - Раскрытие характеристик по клику — доступно только на своей карточке; переброс скрытых характеристик
  - **25 развёрнутых «Ситуаций»** для озвучивания вслух в любой момент партии — кнопка «Ситуация» у хоста
  - **Голосование с таймером**, разблокируется только начиная с 3-го раунда: у всех живых игроков ограниченное время на голос, live-подсчёт по каждому кандидату, обработка ничьей, автозавершение по истечении времени (или досрочно хостом)
  - **Общий чат с таймером обсуждения**: игроки переписываются в реальном времени (через поллинг), хост может запустить общий таймер обсуждения (например, на раунд или на ситуацию), который виден всем в чате и на игровом экране
  - Системные сообщения в чате обо всех ключевых событиях (создание комнаты, вход игрока, катастрофа, раунды, ситуации, голосования, исключения, сброс игры)
  - Случайные события каждого раунда (короткие карточки, всплывают автоматически у всех при переходе в новый раунд)
  - Тёмный постапокалиптический дизайн: текстура шума, виньетка, ржавые акценты, анимации карточек

## Архитектура и данные

### Backend
- **Hono (TypeScript)** на Cloudflare Workers/Pages
- **Cloudflare D1** (SQLite) — единственное хранилище состояния игры; никаких данных в памяти или localStorage сервера
- Игрок на клиенте хранит только `{ code, token }` в localStorage — сам игровой прогресс целиком живёт в D1
- Приватность реализована на уровне SQL-запросов: при каждом запросе `state` сервер сравнивает токен запроса с токеном каждого игрока и либо отдаёт полную карточку (если это «я»), либо только раскрытые поля (`revealed_json`)
- Клиент **поллит** `GET /api/room/:code/state` каждые ~2.5 сек (без WebSocket/Durable Objects — облегчённый стек Cloudflare Pages)

### Таблицы D1 (`migrations/0001_initial_schema.sql`)
- `rooms` — код комнаты, статус (`lobby`/`catastrophe`/`game`/`ended`), катастрофа, бункер, раунд, порог голосования, текущее событие/ситуация, активный таймер (`timer_json`)
- `players` — места комнаты (пустые слоты создаются заранее), токен владельца, все характеристики, `revealed_json` (какие поля раскрыты), флаг исключения
- `chat_messages` — сообщения чата и системные события
- `votes` / `vote_ballots` — сессии голосования и индивидуальные бюллетени (один голос на игрока, можно менять до дедлайна)

### Frontend
- Vanilla JS SPA (`public/static/app.js`) без сборки фреймворка, Axios для запросов, Font Awesome, Google Fonts (Oswald + Rubik)
- Экраны: **Главная** (создать/войти) → **Выбор места** → **Лобби** → **Катастрофа** → **Игра** (карточки + сайдбар с голосованием и чатом)
- Игровой экран рендерит только те данные, которые прислал сервер — чужие нераскрытые характеристики физически отсутствуют в ответе, а не просто скрыты в CSS

## API-эндпоинты (`/api/room/*`)
| Метод | Путь | Описание |
|---|---|---|
| `POST` | `/create` | Тело: `{ playerCount }`. Создаёт комнату с пустыми местами, возвращает `{ code }` |
| `POST` | `/:code/join` | Тело: `{ slot, name }`. Занимает место, возвращает `{ token, isHost, ...state }` |
| `GET` | `/:code/state` | Заголовок `X-Player-Token`. Полное состояние комнаты с приватностью под запрашивающего |
| `POST` | `/:code/start` | Хост. Генерирует катастрофу, бункер и характеристики всем игрокам |
| `POST` | `/:code/reroll-catastrophe` / `/reroll-bunker` | Хост. Перегенерирует катастрофу/бункер |
| `POST` | `/:code/enter-bunker` | Хост. Переводит комнату в статус игры, раунд 1 |
| `POST` | `/:code/reveal` | Тело: `{ field }`. Игрок раскрывает/скрывает своё поле |
| `POST` | `/:code/reroll-attr` | Тело: `{ field }`. Игрок перебрасывает своё ещё нераскрытое поле |
| `POST` | `/:code/exclude` | Тело: `{ playerId }`. Хост исключает/возвращает игрока |
| `POST` | `/:code/next-round` | Тело: `{ seconds }`. Хост. Новый раунд + событие + таймер обсуждения |
| `POST` | `/:code/situation` | Хост. Озвучивает случайную ситуацию |
| `POST` | `/:code/timer/start` / `/timer/cancel` | Хост. Произвольный таймер обсуждения (для чата) |
| `POST` | `/:code/vote/start` | Тело: `{ seconds }`. Хост. Старт голосования (только с порогового раунда) |
| `POST` | `/:code/vote/cast` | Тело: `{ targetPlayerId }`. Голос (можно менять до дедлайна) |
| `POST` | `/:code/vote/finalize` | Хост — в любой момент; любой игрок — после истечения времени |
| `POST` | `/:code/chat` | Тело: `{ text }`. Отправка сообщения в чат |
| `POST` | `/:code/reset` | Хост. Сброс комнаты в лобби с сохранением мест |

## Как пользоваться
1. **Создание комнаты**: на главной вкладке «Создать комнату» выберите количество мест и нажмите «Создать» — получите 5-символьный код.
2. **Приглашение**: передайте код (или скопируйте ссылку-приглашение кнопкой рядом с кодом) остальным игрокам — каждый открывает тот же сайт и вводит код на вкладке «Войти по коду».
3. **Выбор места**: каждый видит сетку мест — свободные кликабельны. Нажав на свободное место, игрок вписывает своё имя и подтверждает — так он «становится» этим персонажем. Первый занявший место автоматически становится хостом.
4. **Лобби**: хост видит кнопку «Начать катастрофу» (активна от 2 занятых мест). Остальные видят статус ожидания.
5. **Катастрофа**: хост может перегенерировать катастрофу/бункер или сразу «Спуститься в бункер».
6. **Игра**: каждый видит свою карточку полностью (все характеристики), а карточки других — только профессию и то, что они сами раскрыли. Раскрытие/переброс доступны только на своей карточке.
7. **Раунды**: хост нажимает «Следующий раунд» — у всех всплывает модалка с событием, счётчик раунда растёт.
8. **Ситуации**: хост нажимает «Ситуация» — у всех появляется модалка с развёрнутым текстом для обсуждения вслух.
9. **Чат**: справа общий чат с сообщениями и системными событиями; хост может запустить таймер обсуждения — он виден всем как обратный отсчёт.
10. **Голосование**: с 3-го раунда хост может запустить голосование с таймером (например, 60 сек). Пока оно активно, каждый живой игрок видит кнопку «Голосовать против» на карточках других и live-подсчёт голосов. По истечении времени голосование автоматически завершается (или хост может завершить раньше) — исключается набравший больше голосов, при ничьей никто не исключается.
11. **Новая игра**: хост может сбросить партию в лобби (с подтверждением) — все остаются на своих местах для новой раздачи.

## Реализованные функции
- ✅ Настоящий мультиплеер: комнаты по коду, каждый игрок со своего устройства
- ✅ Выбор своего места/персонажа в лобби
- ✅ Серверная приватность характеристик (видно только владельцу, пока сам не раскроет)
- ✅ Хост-права на управление игрой (катастрофа, раунды, ситуации, голосование, сброс)
- ✅ Голосование с таймером, разблокируемое с 3-го раунда, автозавершение по истечении времени, обработка ничьей
- ✅ Общий чат с таймером обсуждения и системными сообщениями о событиях игры
- ✅ Генератор персонажей, катастроф, параметров бункера, ситуаций для озвучивания
- ✅ Поллинг состояния комнаты (обновление каждые ~2.5 сек)
- ✅ Адаптивный тёмный дизайн для десктопа и мобильных

## Не реализовано (возможные следующие шаги)
- ❌ WebSocket/Durable Objects для мгновенных обновлений вместо поллинга (сознательно не используется — облегчённый стек Cloudflare Pages)
- ❌ Сохранение истории завершённых партий и статистики по профессиям/катастрофам
- ❌ Печать/экспорт карточек персонажей в PDF
- ❌ Кик игрока хостом до старта игры (сейчас место, если занято, можно освободить только через сброс комнаты)
- ❌ Восстановление сессии, если пользователь потерял токен (например, очистил localStorage) — потребуется отдельный recovery-механизм

### Рекомендации по развитию
1. Добавить более быстрые обновления через Durable Objects/WebSocket, если поллинг раз в 2.5 сек окажется недостаточным.
2. Добавить возможность хосту освобождать/кикать занятое место без полного сброса игры.
3. Реализовать сохранение истории игр в отдельные таблицы D1 для последующей статистики.
4. Добавить звуковой сигнал при истечении таймера голосования/обсуждения.
